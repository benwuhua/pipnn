## 3. Entry Criteria

- `python3 scripts/check_st_readiness.py feature-list.json` returns `READY`
- SRS and Design documents exist
- Required remote configuration is present
- Local build tree is available
- No runtime services are needed because the project is CLI-only

## 4. Exit Criteria

- Applicable regression/integration/E2E/performance/security/compatibility/exploratory checks all have fresh evidence
- `NFR-001`..`NFR-006` all have measured or policy-compliant evidence
- No Critical or Major defects remain open
- RTM coverage is 100% for `FR/NFR/IFR`

## 5. Risk-Based Prioritization

1. Critical path: CLI entrypoint, PiPNN/HNSW flows, benchmark matrix, recall threshold, reproducibility bundle
2. Cross-feature boundaries: CLI -> loader -> runner -> metrics -> result files -> manifest/report linkage
3. Remote compatibility: x86 readiness, authoritative coverage, blocked-state mutation evidence
4. Error handling: unsupported dataset, missing files, numeric parsing, basename output path
5. Residual risk: benchmark timing drift on a shared remote host and future mutation-tool provisioning
