# PiPNN PoC (C++/SIFT1M) Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a C++ PoC that validates PiPNN core pipeline (RBC + leaf kNN + HashPrune), and benchmark against standard `hnswlib` on SIFT1M.

**Architecture:** The PoC is a focused single-binary pipeline with modular components: data loading, graph build primitives, search, and benchmark runner. Unit tests enforce algorithm invariants first (especially HashPrune order-independence), then integration tests validate end-to-end behavior on tiny synthetic data before remote x86 full benchmark.

**Tech Stack:** C++20, CMake, Catch2 (tests), hnswlib (baseline), OpenMP (optional parallel loops), generic-x86-remote scripts for build/benchmark.
